# Attegi 样式指南

欢迎使用 Attegi 主题样式指南！本页面展示了所有可用的格式选项和内容功能。创建文章时可以参考此指南。

## 排版

### 标题

Attegi 支持六级标题，具有优美的排版设计，针对可读性进行了优化。

# 一级标题 (H1)
## 二级标题 (H2)
### 三级标题 (H3)
#### 四级标题 (H4)
##### 五级标题 (H5)
###### 六级标题 (H6)

**最佳实践**：每页只使用一个 H1（文章标题）。H2-H6 会自动出现在目录中。

### 文本格式

您可以通过多种方式格式化文本：

- **粗体文本** 用于强调和关键术语
- *斜体文本* 用于微妙强调或书名
- ~~删除线~~ 用于更正或废弃信息
- `行内代码` 用于变量名、命令或文件名
- [链接](https://attegi.tutuis.me) 使用主题色样式

您还可以组合格式：***粗体和斜体***、**粗体与 `代码`** 等。

### 段落

这是一个标准段落。Attegi 优化了段落间距和行高，以实现最佳可读性。主题使用最佳行长（60-80 个字符）和舒适的行高（1.6-1.8）来减少眼睛疲劳。

保持段落简洁（3-5 句话），段落之间使用空行以提高可读性。用标题、图片和其他视觉元素分隔长文本。

## 列表

### 无序列表

使用项目符号表示无序信息：

- 列表中的第一项
- 包含更多详细信息的第二项
- 第三项
  - 嵌套项目级别 1
  - 另一个嵌套项目

### 有序列表

使用编号列表表示顺序信息：

1. 流程的第一步
2. 带说明的第二步
3. 第三步
   1. 子步骤 A
   2. 子步骤 B
4. 最后一步

- [ ] 未完成任务
- [ ] 另一个未完成任务

## 代码块

### 行内代码

使用行内代码表示短代码片段：`const greeting = "你好，世界！"` 或命令如 `npm install`。

### 带语法高亮的代码块

Attegi 支持 50 多种编程语言的语法高亮：

**JavaScript:**

```javascript
function greet(name) {
  console.log(`你好，${name}！`);
  return true;
}

greet('世界');
```

**Python:**

```python
def calculate_fibonacci(n):
    if n <= 1:
        return n
    return calculate_fibonacci(n-1) + calculate_fibonacci(n-2)

print(calculate_fibonacci(10))
```

**HTML:**

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>示例页面</title>
</head>
<body>
    <h1>你好，世界！</h1>
    <p>这是一个段落。</p>
</body>
</html>
```

**CSS:**

```css
.post-card {
  background: var(--color-background);
  border-radius: 8px;
  transition: transform 0.2s ease;
}

.post-card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 24px rgba(0,0,0,0.15);
}
```

每个代码块都包含复制按钮和语言标签，方便参考。

> ❌ **错误**：这突出显示错误或应避免的事项。

## 内容最佳实践

### 写作技巧

1. **开头要强**：用引人入胜的介绍吸引读者
2. **使用标题**：将内容分成可扫描的部分
3. **添加视觉元素**：每 300-500 字包含图片
4. **保持段落简短**：最多 3-5 句话
5. **使用列表**：使内容可扫描
6. **包含示例**：展示，而不仅仅是讲述
7. **以行动结束**：明确的行动号召或结论

### SEO 技巧

- 使用带关键词的描述性标题
- 为所有图片添加 alt 文本
- 编写引人注目的元描述
- 使用内部链接到相关内容
- 保持 URL 简短且描述性
- 上传前优化图片（最大宽度 1920px）

### 无障碍技巧

- 确保足够的颜色对比度（最小 4.5:1）
- 使用正确的标题层次结构（不要跳级）
- 为图片添加 alt 文本
- 编写描述性链接文本（不要使用"点击这里"）
- 测试键盘导航
- 验证屏幕阅读器兼容性

## 移动优化

Attegi 采用移动优先设计，所有内容都针对智能手机进行了优化：

- **响应式图片**：自动适配每个设备的尺寸
- **触摸友好**：大触摸目标和舒适的间距
- **可读字体**：在所有屏幕上优化的尺寸
- **可折叠目录**：移动端目录变为抽屉
- **快速加载**：延迟加载和优化的资源

在实际移动设备上测试您的内容以确保最佳体验。

## 性能提示

1. **优化图片**：尽可能使用 WebP/AVIF 格式
2. **限制嵌入**：太多会减慢页面加载
3. **使用延迟加载**：图片和嵌入内置此功能
4. **最小化自定义代码**：保持 HTML/CSS 注入最少
5. **定期测试**：使用 Lighthouse 进行性能审计
